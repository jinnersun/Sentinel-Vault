# 代码测试

## 触发时机

功能代码实现并通过代码审查后，调用此技能进行测试验证。

## 测试流程

### 1. 构建验证

首先确保代码可以编译通过：

```powershell
# 后端 Rust 构建
cd src-tauri
cargo check
cargo build

# 前端构建
npm run build
```

### 2. 后端测试

检查 Rust 后端功能：

- [ ] 数据库迁移是否成功执行
- [ ] 新增的 Tauri 命令是否可以调用
- [ ] 数据库表结构是否符合预期
- [ ] SQL 查询是否正确

### 3. 前端测试

检查 React 前端功能：

- [ ] 组件是否可以正常渲染
- [ ] API 调用是否正确
- [ ] 状态管理是否正常工作
- [ ] UI 交互是否符合预期

### 4. 功能验证

根据功能特性进行针对性测试：

**Project Relations 功能测试**：
- [ ] 能否获取项目关联的凭证列表
- [ ] 能否获取未关联的凭证列表
- [ ] 能否创建新的关联关系
- [ ] 能否删除关联关系
- [ ] 关联变更后 UI 是否正确刷新

**数据库迁移测试**：
- [ ] 迁移脚本是否 idempotent（可重复执行）
- [ ] 迁移后数据是否完整
- [ ] 索引是否正确创建

## 输出格式

测试完成后，输出以下格式的报告：


## 常见问题排查

### Rust 编译错误

如果 cargo check 失败：
1. 检查是否有未使用的导入
2. 检查类型是否匹配
3. 检查是否有未实现的 trait

### 前端构建错误

如果 npm run build 失败：
1. 检查 TypeScript 类型错误
2. 检查导入路径是否正确
3. 检查依赖是否正确安装

### 功能不工作

如果功能不工作：
1. 检查后端命令是否在 main.rs 注册
2. 检查前端 API 调用参数是否正确
3. 检查数据库是否有正确的数据
