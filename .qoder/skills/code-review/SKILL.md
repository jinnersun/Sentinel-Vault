# 代码审查

## 触发时机

功能代码实现完成后，自动调用此技能进行审查。

## 审查流程

### 1. 架构合规性检查

检查代码是否符合 V2 架构设计：

- [ ] 凭证 (vault_items) 与项目 (projects) 是否通过 N:N 关联表连接
- [ ] 不应再使用 vault.project_id 强制关联
- [ ] 是否正确使用 credential_project_relations 表
- [ ] 新增的表/字段是否符合 ARCHITECTURE_REDESIGN_V2.md 设计

### 2. 代码质量检查

- [ ] Rust 后端：检查 sqlx 查询是否有 SQL 注入风险（使用参数绑定）
- [ ] React 前端：检查是否正确使用 TypeScript 类型
- [ ] 检查是否有未使用的导入 (unused imports)
- [ ] 检查错误处理是否完整

### 3. 安全检查

- [ ] 凭证密码/密钥是否加密存储
- [ ] 敏感信息是否日志输出
- [ ] 剪贴板操作是否有安全考虑

### 4. API 一致性检查

- [ ] 后端命令是否在 main.rs 的 generate_handler! 中注册
- [ ] 前端 API 封装是否与后端命令匹配
- [ ] 类型定义是否一致

### 5. 构建验证

执行构建检查：

```bash
# 后端检查
cd src-tauri && cargo check

# 前端检查
npm run build
```

## 输出格式

审查完成后，输出以下格式的报告：

```

### 架构合规性

### 代码质量
- [✅/❌] 检查项说明

### 安全问题
- [✅/❌] 检查项说明

### API 一致性
- [✅/❌] 检查项说明

- [✅/❌] cargo check 结果
- [✅/❌] npm run build 结果

### 发现的问题
1. 问题描述 - 严重程度 - 建议修复方案
2. ...

### 总结
- 通过/需要修复
```

## 修复建议

如果发现问题，提供具体的修复方案：
- 文件路径和行号
- 当前代码问题
- 推荐的修复代码
