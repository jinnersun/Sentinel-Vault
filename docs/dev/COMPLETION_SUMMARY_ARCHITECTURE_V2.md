## 🎯 关键改进总结

### V1 架构问题 ❌

```
vault_items.project_id (强制FK关联)
  ↓
凭证必须属于某个项目
  ↓
无法存储独立API KEY、通用凭证等
  ↓
导入功能混乱
  ↓
跨项目共享需要复制
```

### V2 架构解决 ✅

```
vault_items (独立)  +  projects (独立)  →  credential_project_relations (N:N)
  ↓
凭证可独立存在，也可关联多个项目
  ↓
支持独立API KEY、通用凭证
  ↓
导入有独立历史表管理
  ↓
跨项目共享原生支持
  ↓
关系类型灵活标记 (primary/associated/shared)
```

### 具体场景改进

| 场景 | V1 方式 | V2 方式 |
|------|---------|--------|
| 添加通用 API KEY | ❌ 创建虚拟项目 | ✅ 直接创建，无需项目 |
| Chrome 导入 | ❌ 混在凭证中 | ✅ 独立历史表管理 |
| 跨项目共享 | ❌ 复制凭证 | ✅ 一个凭证，多项目关联 |
| 项目为空 | ⚠️ 未定义 | ✅ 完全支持 |
| UI 导航 | 🟡 复杂 | ✅ 清晰多维度 |
