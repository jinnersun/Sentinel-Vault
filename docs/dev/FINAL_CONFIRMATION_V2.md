## 💡 核心差异理解

### 问题 1: API KEY 存储位置

**V1**: API KEY 必须关联某个项目
```
问题: 创建虚拟项目
UI混乱，不优雅
```

**V2**: API KEY 可独立存在
```
优势: 无需虚拟项目
干净清爽，专业
```

---

### 问题 2: 跨项目凭证共享

**V1**: 两个项目需要同一个数据库
```
方案1: 复制凭证 (不一致风险)
方案2: 虚拟共享项目 (复杂)
```

**V2**: 一个凭证关联多个项目  
```
一个事实来源
修改一次，处处生效
```

---

### 问题 3: 导入历史管理

**V1**: Chrome 导入混在凭证中
```
无法追踪导入历史
无法看到冲突情况
无法审计
```

**V2**: 独立的导入历史表
```
完整的导入记录
冲突检测和处理
可追踪和可回滚
```
